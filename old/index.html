Deterministic Engine Features

Built up piece by piece — every system component maps directly to strict, isolated language rules.

State-Machine Lifecycles

Control variables natively using keyword statements like archive, restore, reset, and delete.

Hard Type Isolation

Integers (long long) and Floats (double) are kept distinct. 5 == 5.0 evaluates to false.

Explicit Closures

Functions see nothing outside their scope by default. Pass names to [captures] to grant explicit mutating access.

Type & Interface Blueprints

Validate object literals structurally with strict signature matching or flexible fallbacks to null.

Reference-Counted Arrays

Supports literal growable lists or bounded, fixed-size layouts that reject memory mutation.

File System Modules

Dynamic compile-time code composition using a structural @import "file.swiq"; pipeline directive.

State Manipulation

Native Lifecycle Management

Swiq embeds memory and lifecycle operations directly into statements. Instead of waiting for loose garbage collections, you dictate state flags natively.

Using archive safely shifts an identifier from the active runtime context into background memory preservation, locking it out of mutations. Bring it back instantly using restore, or trigger reset to roll values back to their initial declaration block.

set var token = "0x7F";

archive token;
# log(token); ❌ Exception: variable is archived.

restore token;
log(token);
0x7F
set interface NetworkConfig = {
    port: Number = 8080,
    bitrate: Number<integerOnly>
};

set var primarySocket = { bitrate: 9600 }<NetworkConfig>;
log(primarySocket.port);
8080
Structural Verification

Type vs. Interface Blueprints

Construct complex, verified object structures using explicit colon templates. Fields enforce matching base types like Number, String, or Boolean, along with targeted validation modifiers like <integerOnly>.

Choose your contract enforcement: type definitions act as open structures where missing parameters collapse down cleanly to null, while strict interface definitions drop full runtime exceptions if mandatory initializations are omitted.

Try it yourself

Clone the repo, build with CMake, run your first .swiq file.

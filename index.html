Explicit closures

The Swiq way

Closures, but you have to ask first

Most languages let a function quietly see every variable around it. Swiq flips that: functions are isolated by default, and you list exactly which outer variable a function is allowed to touch.

Listed variables get full read-and-write access — change them inside the function, and the change is still there once it returns. Anything you didn't list stays invisible.

set var counter = 0;

func increment()[counter] {
    set counter = counter + 1;
    return counter;
}

log(increment());
log(increment());
log(counter);
1
2
2
set var arr = [10, 20, 30];
push(arr, 40);

set var sum = 0;
for (set var i = 0; i < len(arr); set i = i + 1) {
    set sum = sum + arr[i];
}

log(sum);
100
Loops & arrays

C-style for loops, real array mutation

Arrays have reference semantics — push into one inside a loop or a function, and every variable pointing at it sees the update. Combined with a familiar three-clause for, you get the building blocks for actual programs, not just toy scripts.

Try it yourself

Clone the repo, build with CMake, run your first .swiq file.

$ git clone https://github.com/bananakitssu/swiq.git && cd swiq mkdir build && cd build cmake .. make ./swiq ../examples/hello.swiq
